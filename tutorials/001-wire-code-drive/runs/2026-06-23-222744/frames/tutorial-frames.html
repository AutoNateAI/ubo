Step 6

Paste firmware

The code turns pin states into robot behaviors: forward, reverse, pivot, stop.

Check serial output before trusting movement.
robot_car_drive.inoCodeRun
void driveForward(int speed) {
  setLeftMotor(true, speed);
  setRightMotor(true, speed);
  logState("forward", speed, speed);
}

void pivotRight(int speed) {
  setLeftMotor(true, speed);
  setRightMotor(false, speed);
  logState("pivot-right", speed, speed);
}

void stopRobot() {
  digitalWrite(LEFT_FWD, LOW);
  digitalWrite(LEFT_REV, LOW);
  digitalWrite(RIGHT_FWD, LOW);
  digitalWrite(RIGHT_REV, LOW);
  analogWrite(LEFT_PWM, 0);
  analogWrite(RIGHT_PWM, 0);
}
Step 7

Run and verify

Do not just press Run. Compare serial logs to the motor-driver state.

Pass only when code, pins, and behavior agree.
Run checklistSerialRunning
1. Serial says forward; both sides drive forward.
2. Serial says pivot-right; left forward and right reverse.
3. Serial says pivot-left; left reverse and right forward.
4. Serial says reverse; both sides reverse.
5. Serial says stop; PWM is zero and direction pins are LOW.
