# Learning Video Workflow

Use `tutorials/NNN-name` as the source of truth for each learning video.

## Package Files

- `README.md`: full learner-facing walkthrough
- `outline.md`: beats for the recording
- `voiceover.md`: narration script
- `take-notes.md`: mistakes, retakes, timestamps, and improvement notes
- `screenshots/`: still captures for thumbnails or overlays
- `exports/`: simulator exports or source artifacts
- `firmware/`: code used in the lesson

## Recording Pattern

1. Start with the behavior: show what the robot should do.
2. Show the wiring before writing code.
3. Explain the part that makes the behavior possible.
4. Add firmware and explain the pin map.
5. Run the simulator.
6. Compare expected behavior to actual behavior.
7. Debug one thing clearly.
8. End with the next challenge.

## Repurposing For YouTube

Prepare upload material in markdown first:

- title
- description
- chapter timestamps
- pinned comment
- thumbnail idea
- short-form cutdowns

Do not publish externally from automation until the exact account, destination, title, description, and video file are confirmed.
